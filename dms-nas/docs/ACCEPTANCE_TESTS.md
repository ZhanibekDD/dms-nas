# Acceptance Tests (Критерии готовности)

Каждый тест описывает сценарий end-to-end, который должен работать без ошибок.

---

## AT-01 Upload — загрузка PDF прорабом

**Предусловие**: Прораб зарегистрирован в боте, объект существует на NAS.

**Шаги**:
1. `/start` → получить главное меню
2. `📤 Загрузить документ`
3. Выбрать объект «Днепр»
4. Выбрать тип «Акт»
5. Участок «фундамент»
6. Отправить PDF-файл

**Ожидаемый результат**:
- ✅ Файл появляется на NAS: `/Днепр/_INBOX/Акт/<filename>`
- ✅ Запись в `uploads_log` со статусом `pending`
- ✅ Бот отвечает «Загружено!» с путём

## AT-03 Approve — утверждение документа
