## Flow 1 — Загрузка документа (прораб)

1. `/menu` → `📤 Загрузить документ`
2. Выбор объекта из списка NAS
3. Выбор типа: Сертификат / ТТН / Акт / Протокол / ФотоОтчет / Другое
4. Участок/раздел (опционально, «-» = пропустить)
5. Отправить файл/фото/видео
6. Бот загружает на `/{Объект}/_INBOX/{Тип}/filename`
7. Запись в `uploads_log`, статус `pending`

---

## Flow 2 — Просмотр и скачивание (все роли)

1. `/menu` → `📂 Найти документ`
2. Навигация по дереву NAS через inline-кнопки
3. Нажать на файл → бот скачивает с NAS и отправляет в чат

---

## Flow 3 — Утверждение (ПТО/ТБ/admin)

1. `✅ На проверке` → список документов со статусом `pending`
2. Карточка: файл, объект, тип, загрузивший, дата
3. Кнопки: Утвердить / Отклонить / Скачать / Следующий
4. **Утвердить**: копия в `_APPROVED/`, статус → `approved`, уведомление прорабу
5. **Отклонить**: ввод причины → копия в `_REJECTED/`, статус → `rejected`, уведомление

---

## Flow 4 — Сроки и напоминания

1. `⏰ Сроки` → список активных сроков с цветовой индикацией
2. `➕ Добавить срок` → название → объект → дата (ГГГГ-ММ-ДД)
3. Ежедневно в 09:00 планировщик проверяет сроки:
   - T-30 дней → 🔵 уведомление
   - T-7 дней → 🟡 уведомление
   - T-1 день → 🟠 уведомление
   - Просрочено → 🔴 уведомление каждый день

---

## Flow 5 — Фотоотчёт

1. `📸 Фотоотчёт` → выбор объекта → выбор чек-листа
2. Пошагово: бот просит фото по каждому пункту
3. «⏭ Пропустить» — пропустить пункт
4. Файлы именуются `01_НазваниеПункта.jpg`, `02_...` и т.д.
5. Сохраняются в `/{Объект}/ФотоОтчет/{ГГГГ-ММ-ДД}/`

---

## Flow 6 — Пакет документов

1. `📦 Пакет документов` → объект → период (например: `2025-12`) → типы
2. Бот собирает файлы из `_APPROVED/` (или `_INBOX/` если нет approved)
3. Формирует ZIP до 50 МБ + `summary.md`
4. Сохраняет ZIP на NAS в `/_PACKAGES/`, отправляет в чат

---

## Flow 7 — Финансы (бухгалтер)

1. `💰 Финансы` → список документов
2. `➕ Добавить финдок` → объект → тип → файл → сумма → контрагент
3. Документ создаётся со статусом `черновик`
4. `🔄 Изменить статус` → ID документа → новый статус (по матрице переходов)
5. `📊 Экспорт CSV` → скачать CSV с BOM (открывается в Excel)

**Матрица переходов:**
```
черновик → на_проверке  (buh/admin)
на_проверке → утверждён  (pto/admin)
на_проверке → отклонён   (pto/admin)
утверждён → оплачен      (buh/admin)
отклонён → черновик      (buh/admin)
```

---

## Flow 8 — Создать объект

1. `🏗️ Создать объект` → ввод имени → подтверждение
2. Создаётся полная структура папок на NAS (25+ папок)

---

## Flow 9 — Web Admin

- Вход: http://сервер:8000/admin — login `admin`, password из config
- **Дашборд**: `/` — KPI карточки + последние загрузки
- **Документы**: фильтры по статусу/типу/объекту, approve/reject через actions, скачать
- **Сроки**: список с цветовой индикацией, архивировать
- **Финансы**: статусы, CSV экспорт через action, скачать с NAS
- **Проблемы**: список, закрыть через action
- **Аудит**: read-only журнал всех действий
- **NAS Proxy**: `/nas-proxy/?path=/Объект/...` — сервер качает с NAS и отдаёт браузеру
